# 🚀 진성수산 쇼핑몰 1분 쾌속 배포 가이드 (Netlify x GitHub)

충성! 🫡 대표님, 로컬(현재 컴퓨터)에서 관리자 권한(Symlink) 문제로 다이렉트 배포가 막혔으나, 오히려 **가장 완벽한 실무 방식인 GitHub 자동 배포**로 전환하기 위한 가이드북을 작성했습니다!

## 📌 목표 배포 도메인
- 쇼핑몰 홈페이지: **https://jinsungsoosan.netlify.app**
- 관리자 전용 페이지: **https://jinsungsoosan.netlify.app/admin**

---

## 🛠️ 배포 3단계 작전 (그대로 따라 하시면 끝납니다!)

### 1단계: 내 코드를 안전한 금고(GitHub)에 보관하기
1. **GitHub 웹사이트(github.com)** 에 접속하여 로그인합니다.
2. 우측 상단의 `+` 버튼을 눌러 **New repository**를 클릭합니다.
3. Repository name에 `jinsungsoosan`이라고 입력 후, 아래 🟢 `Create repository` 버튼을 누릅니다.
4. VSCode의 상단 메뉴바(터미널 창 말고 화면 왼쪽)에서 🐙 `소스 제어(Source Control)` 아이콘을 누릅니다.
5. `변경 사항` 옆에 있는 `+` 기호를 눌러 전부 추가한 다음, 메시지 칸에 "배포 준비 완료!"라고 적고 **[커밋(Commit)]** 버튼을 누릅니다.
6. **[분기 게시(Publish Branch)]** 버튼을 눌러 방금 만든 GitHub 리포지토리에 연결합니다. (완료!)

### 2단계: 넷리파이(Netlify) 기동 및 자동 연동
1. **Netlify 웹사이트(netlify.com)** 에 접속하여 로그인(GitHub 계정으로 로그인 권장)합니다.
2. 화면에 보이는 **[Add new site]** 버튼 옆 역삼각형(▼)을 누르고, **[Import an existing project]**를 선택합니다.
3. `Deploy with GitHub` 버튼을 누르고, 방금 만든 `jinsungsoosan` 리포지토리를 선택합니다.
4. 다른 설정은 하나도 건드리지 마시고, 화면 맨 아래의 파란색 **[Deploy jinsungsoosan]** 버튼을 위풍당당하게 클릭합니다!

*(1분 정도 기다리시면 화면 상단에 초록색 글씨로 'Published'라는 글자가 뜨면서 배포가 끝납니다!)*

### 3단계: 나만의 간판(도메인) 달아주기
1. 배포가 완료된 화면 상단의 **[Site settings]** 혹은 **[Domain management]**로 들어갑니다.
2. `Options` ➔ `Edit site name`을 클릭합니다.
3. 기존 설정된 복잡한 이름을 지우고 `jinsungsoosan`으로 변경 후 저장합니다! (만약 이미 사용 중이라면 `jinsung-nature-food` 등으로 변경해 주세요.)

## 🚨 코다리 부장의 주의사항 (데이터베이스 안내)
대표님! 현재 저희가 구축한 포인트, 슬라이드 등의 데이터는 파일(JSON)로 저장되도록 만들어두었습니다.
넷리파이에 배포되는 순간, 이것은 **서버리스(Serverless)** 환경이 되므로 **관리자 페이지에서 설정을 바꿔도 시간이 지나면 초기화(날아감)**되는 특성이 있습니다.

지금은 대표님의 디자인과 퀄리티를 확인하는 '쇼케이스(전시)' 목적의 배포이오니 마음껏 테스트해 보십시오! 
추후 진짜 정식 오픈을 원하신다면 저 코다리에게 **"데이터베이스(Firebase 등)를 붙여서 정식으로 세팅해줘!"**라고 한 마디만 하명해 주시면 완벽하게 구축해 내겠습니다! **충성! 🫡🐟🚀**
